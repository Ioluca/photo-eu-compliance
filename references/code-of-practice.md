# GPAI Code of Practice — Copyright Chapter Reference

## Last updated: 2025-08-02 (final version published July 10, 2025)

## Source
General-Purpose AI Code of Practice, European Commission
Published: July 10, 2025 | In force: August 2, 2025
Status: Living document — will be revised periodically

---

## Overview

The Code of Practice implements AI Act Art. 53-55 with practical requirements
for general-purpose AI model providers. The copyright chapter is the most
relevant for photographers.

**Key point:** This Code applies to AI PROVIDERS (OpenAI, Google, Meta,
Stability AI, Adobe, etc.), not to photographers. But it defines the
mechanisms photographers can use to protect their work.

---

## What Providers Must Do

### 1. Publish a Copyright Policy
Every GPAI provider must publish a clear, accessible copyright policy covering:
- How they handle opt-outs from rights holders
- What data sources they use for training
- How to submit complaints about unauthorized use

### 2. Respect Opt-Outs
Providers must implement technical measures to respect rights holders'
reservations under Copyright Directive Art. 4(3). This includes:
- Honoring robots.txt directives
- Processing opt-out requests
- Maintaining records of opt-out compliance

### 3. Document Training Data
Providers must create and publish summaries of training data content,
using the template published by the European Commission on July 24, 2025.

### 4. Offer Complaint Mechanisms
Providers must operate an accessible, time-bound complaint mechanism
for rights holders who believe their content was used without authorization.

---

## What This Means for Photographers

### Your rights
1. You can demand any GPAI provider tell you their copyright policy
2. You can submit opt-out requests and expect them to be processed
3. You can file complaints if you believe your images were used
4. You can check public training data summaries for your content

### Practical steps
1. **Opt out proactively:** Use robots.txt, meta tags, platform settings
2. **Send formal opt-out letters:** Reference Art. 4(3) and the Code of Practice
3. **Monitor compliance:** Check provider summaries when published
4. **File complaints:** Use provider mechanisms if you find unauthorized use

### Limitations
- The Code is voluntary (providers sign on) but compliance is expected
  for demonstrating AI Act compliance
- Enforcement is through the European AI Office, not through courts directly
- The "living document" nature means mechanisms will evolve
- Retroactive opt-out (for images already in training data) remains unresolved
